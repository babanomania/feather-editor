## 2. `src/index.jsx` ŌĆö Core Editor

Uses `useRef` + browser `execCommand` + CSS custom properties. Props:
- `markdown` ŌĆö controlled value (Markdown string)
- `onChange(md)` ŌĆö fires with Markdown string on every input
- `theme` ŌĆö optional string (e.g. `"dark"`) ŌĆö adds class `feather-theme-dark` to wrapper
- `className`, `style` ŌĆö passthrough

```jsx
import { useRef } from "react";
import { toHtml, toMd } from "./markdown.js";

const cmd = (c, v) => document.execCommand(c, false, v);

export default function Editor({ markdown = "", onChange, theme, className = "", style }) {
  const ref = useRef(null);
  const sync = () => onChange?.(toMd(ref.current.innerHTML));
  const B = ({ c, v, label }) => (
    <button onMouseDown={(e) => { e.preventDefault(); cmd(c, v); sync(); }}>{label}</button>
  );
  const cls = ["feather-editor", theme && `feather-theme-${theme}`, className].filter(Boolean).join(" ");
  return (
    <div className={cls} style={style}>
      <div className="feather-bar">
        <B c="bold" label="B" />
        <B c="italic" label="I" />
        <B c="underline" label="U" />
        <B c="formatBlock" v="<h2>" label="H2" />
        <B c="insertUnorderedList" label="ŌĆó List" />
      </div>
      <div ref={ref} className="feather-content" contentEditable suppressContentEditableWarning
           onInput={sync} dangerouslySetInnerHTML={{ __html: toHtml(markdown) }} />
    </div>
  );
}
```

## 3. `src/markdown.js` ŌĆö Markdown Round-Trip

Hand-written; no external parser. Handles: `**bold**`, `*italic*`, `## heading`, `- list`, `[text](url)`, paragraphs.

**`toHtml(md)`** ŌĆö Markdown ŌåÆ HTML (for rendering into contenteditable):
- Split by `\n\n` into blocks
- Per block: detect `## ` ŌåÆ `<h2>`, `- ` lines ŌåÆ `<ul><li>`, else `<p>`
- Inline: replace `**x**`ŌåÆ`<strong>`, `*x*`ŌåÆ`<em>`, `[t](u)`ŌåÆ`<a href="u">t</a>`

**`toMd(html)`** ŌĆö HTML ŌåÆ Markdown (for onChange output):
- Walk DOM (via `DOMParser`) serializing nodes
- `STRONG/B` ŌåÆ `**...**`, `EM/I` ŌåÆ `*...*`, `H2` ŌåÆ `## `, `UL/LI` ŌåÆ `- `, `A` ŌåÆ `[text](href)`, `P` ŌåÆ paragraph

## 4. `build.js` ŌĆö esbuild

```js
const esbuild = require("esbuild");
const shared = { entryPoints: ["src/index.jsx"], bundle: true, external: ["react"], minify: true };
Promise.all([
  esbuild.build({ ...shared, format: "esm", outfile: "dist/index.esm.js" }),
  esbuild.build({ ...shared, format: "cjs", outfile: "dist/index.js" }),
]).catch(() => process.exit(1));
```

CSS theme files require no build step ŌĆö they ship from `themes/` directly.
